# DVCA Research

本目录用于记录 DVCA 项目的研究问题、假设、发现和结论。

这里不是案例数据库，也不是截图仓库。这里记录的是“为什么会这样”的研究过程。

## 目录结构

```text
research/
├── questions/
├── hypotheses/
├── findings/
└── conclusions/
```

## 使用方式

### 1. questions

记录研究问题。

例子：

```text
BTC 在趋势行情里为什么 TC-L 很多，但 LONG 很少？
```

### 2. hypotheses

记录初步假设。

例子：

```text
可能是 BTC 趋势段回踩浅，满足 TC 条件，但没有形成足够清晰的 Pivot 背离 Zone。
```

### 3. findings

记录收集 20-30 个案例后的观察结果。

例子：

```text
BTC 30m 趋势段中，TC-L 出现 18 次，LONG 只出现 3 次。
```

### 4. conclusions

记录最终结论和是否需要修改指标。

例子：

```text
不建议为了 BTC 强行增加 LONG；应把 BTC 趋势行情单独归入 TC 行为研究。
```

## 研究原则

- 先记录问题，不急着改指标。
- 先收集真实案例，不凭感觉下结论。
- 先统计，再决定是否升级 DVCA。
- 每个结论都应该能追溯到截图、案例或统计结果。
