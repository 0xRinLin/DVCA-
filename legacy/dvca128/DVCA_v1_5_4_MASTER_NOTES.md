# DVCA v1.5.4 Main Version Notes

主版本：

`DVCA_v1_5_4_Decoupled_Safety_Patch.pine`

当前结论：

`v1.5.4` 作为主版本保留。不要继续沿着 `1.6.x` 或极简显示版方向改核心逻辑。

可选保守版：

`DVCA_v1_5_4R_Conservative.pine`

`v1.5.4R` 不是替代主版本，而是用于测试“强趋势中反向 Zone 更保守”的旁支。若 R 版导致高周期结构提示过少，直接退回 `v1.5.4`。

`v1.5.4R` 只改三件事：

- 强上涨背景中的 `S-ZONE` 必须有失败确认才升级为正式 Zone。
- 强下跌背景中的 `L-ZONE` 必须有失败确认才升级为正式 Zone。
- `ZC-L / ZC-S` 默认关闭，并新增 `Pivot标签显示位置 = Confirm Bar / Pivot Bar`。

## 定位

`30m / 1H / 4H / 1D` 在 Auto 模式下是 HTF Context。

高周期用途：

- 判断顶部/底部结构区域
- 判断追多/追空风险
- 给 5m / 15m 执行信号做过滤
- 不追求频繁 LONG / SHORT

低周期用途：

- `5m / 15m` 找执行信号
- `1m` 做更细的进场观察

## 推荐多周期组合

- `1H`: 大背景
- `30m`: 结构区域
- `15m`: 走势转弱/转强确认
- `5m`: 执行信号
- `1m`: 微观观察，不单独决定方向

## 信号解读

`S-ZONE`:

高周期顶部风险区。不要盲目追多；小周期 SHORT 更值得重视。

`L-ZONE`:

高周期底部观察区。不要盲目追空；小周期 LONG 更值得重视。

`EXH-S`:

多头可能衰竭，追多风险升高。

`EXH-L`:

空头可能衰竭，追空风险升高。

`LONG / SHORT`:

执行周期的正式信号。默认只有它们会进入持仓状态并等待 PB。

`PB-L / PB-S`:

突破后的回踩确认。

`XL / XS`:

简单离场/减仓提示。

## 已知失准来源

这些不是代码 bug，而是结构类指标的天然边界：

- `L-ZONE / S-ZONE` 是区域，不是进场命令。强趋势或急拉急跌中，Zone 可以连续失效。
- `ZC-L / ZC-S` 是 pivot 被确认的位置，不等于突破确认，也不等于 LONG / SHORT。
- 背离信号在单边趋势中会提前出现，价格可以继续沿原方向走很远。
- RSI 超买/超卖不能单独当反转，强趋势中会长时间钝化。
- MACD/EMA 类动能确认有滞后，急速拉升或杀跌时会慢半拍。
- 1m / 5m 的噪音很大，必须服从 15m / 30m / 1H 的结构背景。
- 30m / 1H 的 Zone 更适合做风险过滤，不适合直接当执行信号。
- 历史图上的 Zone 使用 pivot offset 显示在极值附近，但真实可见时间是在 `pR` 根 K 之后。
- 被后续走势 invalid 的历史标签不会自动消失，所以需要结合 HUD 的 `Rj` 和后续结构判断。

## 读图纪律

- 高周期出现 `S-ZONE / EXH-S`：优先理解为不追多，不等于立刻做空。
- 高周期出现 `L-ZONE / EXH-L`：优先理解为不追空，不等于立刻做多。
- 真正执行优先看 `LONG / SHORT`，其次看 `PB-L / PB-S`。
- `TC-L / TC-S` 是趋势延续，容易追价；默认可以先关闭显示。
- `E-L / E-S` 是提前观察，不能当正式信号。
- 一个信号如果随后被结构突破反向打穿，要按失败处理，而不是继续相信原标签。

## 禁止事项

- 不要把显示开关重新接回交易状态机。
- 不要让 alert 再受 showXXX 控制。
- 不要默认打开 RC / TC-W / LATE-W。
- 不要把 30m / 1H 改成频繁执行信号图。
- 不要继续为了“更干净”牺牲 Zone、背离连线和 EXH 的结构感。
